# 목양담당 시스템 기준 이음앱 검토

기준 문서: `01. Ministry/01. 이음채플/04. Formation & Care/목양담당 운영 시스템 설계.md`

## 결론

현재 이음앱은 방문자 카드, 새가족 단계 관리, 멤버 관리, 순 배정, 출석 관리의 기반은 갖추고 있다.

하지만 목양담당 시스템의 핵심인 "큐", "담당자 인계", "다음 조치", "연결 완료 확인", "돌봄 필요 판단"은 아직 앱의 1급 기능으로 존재하지 않는다.

현재 앱은 아래 흐름에 가깝다.

```text
방문자 카드
-> members.status = new_family
-> new_family step 관리
-> step 3 완료 시 adjusting
-> 3개월 후 attending 자동 전환
```

설계한 목양담당 시스템은 아래 흐름이어야 한다.

```text
방문자 카드
-> 웰컴 큐
-> 웰컴담당 첫 연락
-> 새가족 전환 판단
-> 새가족담당 교육 진행
-> 목양담당 연결 큐
-> 연결담당/다락방담당 실행
-> 목양담당 연결 완료 확인
-> 정기 목양/돌봄 큐
```

따라서 이음앱은 "새가족 관리"에서 "목양 운영 시스템"으로 확장되어야 한다.

## 현재 구현된 것

### 1. 방문자 카드 입력

`/visitor-card`에서 외부 방문자가 직접 정보를 입력할 수 있다.

현재 저장 방식:
- `members`에 바로 생성
- `status`는 `new_family`
- `new_family`에도 바로 생성

좋은 점:
- 외부 인풋이 앱으로 들어오는 입구가 이미 있다.
- 이름, 연락처, 카카오, 학교/직장, 생년월일, 성별, 세례 여부, 이전 교회 정보를 받는다.

부족한 점:
- 방문자 카드가 "웰컴 큐"로 생성되지 않는다.
- 웰컴담당이 1주 이내 연락해야 하는 다음 조치가 생성되지 않는다.
- 방문자와 새가족이 분리되지 않는다.

### 2. 새가족 단계 관리

`/new-family`에서 새가족을 단계별로 관리한다.

현재 단계:
- 1주차 방문
- 2주차 교육
- 3주차 교육

좋은 점:
- 교육 진행 상태를 볼 수 있다.
- 2주 이상 단계 변화가 없는 사람을 화면에서 보여준다.
- 담당자(`assigned_to`)를 둘 수 있다.

부족한 점:
- 담당자가 웰컴담당인지, 새가족담당인지, 목양담당인지 구분되지 않는다.
- 새가족교육 안내 여부, 연락 여부, 참석 예정일, 다음 조치가 별도 구조로 없다.
- 새가족담당에서 목양담당으로 인계하는 액션이 없다.

### 3. 멤버/순/사역팀 연결 표시

멤버 화면에서 현재 순과 사역팀을 볼 수 있다.

좋은 점:
- 순 배정 정보가 있다.
- 사역팀 연결 정보가 있다.
- 연결 완료 판단에 필요한 일부 데이터가 이미 있다.

부족한 점:
- "연결 진행 중"과 "연결 완료"가 별도 상태로 관리되지 않는다.
- 연결 완료를 목양담당이 확인하는 액션이 없다.
- 순 배정, 관계 연결, 섬김 연결을 하나의 목양 상태로 묶어 보여주지 않는다.

### 4. 출석 관리

순별 출석 기록 기능이 있다.

좋은 점:
- 장기 미출석 큐를 만들 수 있는 기반 데이터가 있다.

부족한 점:
- 최근 4주 미출석자를 자동으로 목양 큐에 올리지 않는다.
- 안부 연락, 리더 공유, 복귀/이동/종료 판단 흐름이 없다.

## 주요 문제

### P1. 방문자 카드가 웰컴 큐가 아니라 곧바로 새가족이 된다

방문자 카드 제출 시 `members.status = new_family`로 저장되고 `new_family` 레코드도 즉시 생성된다.

문제:
- 웰컴담당의 독립 역할이 앱에 없다.
- 첫 방문자와 새가족교육 대상자가 같은 목록에 섞인다.
- 방문자에게 감사 연락을 했는지 추적할 수 없다.
- 새가족 전환 판단이 사라진다.

필요한 변경:
- 방문자 카드를 `visitor_cards` 또는 `care_people`의 방문자 상태로 먼저 저장한다.
- 웰컴 큐에 `감사 연락` 조치를 자동 생성한다.
- 웰컴담당이 `새가족 전환` 버튼을 누를 때 `members`와 `new_family`로 넘긴다.

### P1. 목양 큐 테이블이 없다

현재 앱에는 "누가, 언제, 무엇을 해야 하는지"를 표현하는 테이블이 없다.

필요한 개념:
- 큐 유형: 웰컴, 새가족, 연결, 돌봄, 장기 미출석
- 대상자
- 담당 역할
- 담당자
- 다음 조치
- 기한
- 완료 여부
- 상태

없을 때 생기는 문제:
- 목양담당이 전체 흐름을 앱에서 운영할 수 없다.
- 웰컴담당/새가족담당/다락방담당/돌봄담당에게 일을 배정할 수 없다.
- 완료되지 않은 조치를 추적할 수 없다.

### P1. 새가족교육 완료 후 연결 큐로 넘어가지 않는다

현재는 3주차 교육 완료 시 `members.status`가 `adjusting`으로 바뀐다.

문제:
- 목양담당에게 인계되지 않는다.
- 연결담당 또는 다락방담당에게 다음 조치가 생성되지 않는다.
- 소속 의사 확인, 순 연결, 섬김 연결, 관계 연결 여부가 별도 확인되지 않는다.

필요한 변경:
- 3주차 교육 완료 시 `연결 진행 중` 상태로 전환한다.
- 동시에 연결 큐를 생성한다.
- 다음 조치 예: `소속 의사 확인`, `순 연결 제안`, `관계 연결자 지정`.

### P1. 3개월 후 자동 attending 전환은 목양 시스템과 충돌한다

현재 앱은 `adjusting` 상태가 3개월 지나면 자동으로 `attending`으로 바뀐다.

문제:
- 시간이 지나면 연결 완료처럼 보인다.
- 목양담당의 확인 없이 성도 상태가 바뀐다.
- 연결 실패자를 발견하기 어렵다.

필요한 변경:
- 3개월 자동 전환을 중단한다.
- `연결 완료` 버튼 또는 큐 완료 액션으로만 `attending`에 해당하는 상태로 넘긴다.

### P2. "담당자"의 의미가 하나로 뭉쳐 있다

`new_family.assigned_to`는 있지만 이것이 웰컴담당, 새가족담당, 목양담당, 연결담당 중 누구인지 구분되지 않는다.

필요한 변경:
- 역할 기반 담당을 분리한다.
- 예: `welcome_owner_id`, `new_family_owner_id`, `care_owner_id` 또는 별도 assignment 테이블.

추천:
- `care_assignments` 테이블을 만들어 역할별 담당을 유연하게 관리한다.

### P2. 돌봄 필요와 장기 미출석 큐가 없다

출석 데이터는 있지만, 최근 4주 미출석자가 자동으로 큐에 올라오지 않는다.

필요한 변경:
- 최근 4주 출석 기록이 없는 성도 조회
- 장기 미출석 큐 자동 생성
- 안부 연락, 리더 공유, 상태 재분류 액션 추가

### P2. 목양 로그가 없다

현재 상태 변경 로그는 있지만, 목양적 조치 로그는 없다.

필요한 변경:
- 연락 완료
- 식사 초대
- 교육 안내
- 순 연결 제안
- 리더 공유
- 목회자 연결

위 조치를 짧게 기록할 수 있어야 한다.

민감 정보 보호를 위해 공개 범위도 필요하다.

## 추천 데이터 모델

### visitor_cards

필드:
- id
- name
- phone
- kakao_id
- gender
- birth_date

- previous_church
- submitted_at

필드:
- id

- person_id

- assigned_role
- assigned_to
- due_date
- status: `open`, `in_progress`, `done`, `blocked`, `closed`
- note
- created_at
- completed_at

### member_care

멤버의 목양 상태를 저장한다.

필드:
- member_id

- connection_status: `not_started`, `in_progress`, `completed`, `reconnect_needed`

- care_owner_id
- membership_confirmed_at
- new_family_completed_at
- connection_completed_at

### care_logs

사람별 목양 로그다.

필드:
- id
- member_id
- visitor_card_id
- task_id
- author_id
- log_type
- visibility: `care_team`, `leadership`, `pastor_only`
- body
- created_at

### 2. 웰컴 큐

보여줄 것:

### 3. 새가족 큐

## 단계별 구현안

### 1단계: 최소 변경으로 목양 시스템 느낌 만들기

- `adjusting` 라벨을 "연결 진행 중"으로 변경
- 3개월 자동 `attending` 전환 제거
- 새가족 완료 섹션 이름을 "연결 필요"로 변경
- 새가족 완료자에 순 연결 여부 표시
- 연결 완료 버튼 추가

### 2단계: care_tasks 추가

- 목양 큐 테이블 생성
- 방문자 카드 제출 시 웰컴 task 생성
- 새가족교육 완료 시 연결 task 생성
- 4주 미출석 task 생성

### 3단계: visitor_cards 분리

- 방문자 카드를 `members`에 바로 넣지 않음
- 웰컴담당이 새가족 전환할 때 members/new_family 생성
- 방문자 종료/보류 상태 관리

### 4단계: 목양 로그와 권한

- care_logs 추가
- 공개 범위 추가
- 웰컴담당/새가족담당/다락방담당/돌봄담당 role 추가

## 우선순위

가장 먼저 고칠 것:

1. `adjusting` 자동 3개월 전환 제거
2. `adjusting` 표시명을 "연결 진행 중"으로 변경
3. 새가족 완료자 섹션을 "연결 필요"로 바꾸기
4. 연결 완료를 수동 처리하는 액션 추가
5. 방문자 카드 제출 후 웰컴 큐 생성

이 순서가 좋은 이유:
- 기존 데이터 모델을 크게 깨지 않는다.
- 새가족교육 완료 후 목양담당이 일할 자리가 생긴다.
- 방문자 분리는 나중에 해도 되지만, 자동 전환은 빨리 멈추는 것이 좋다.
